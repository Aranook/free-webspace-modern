Attention : GitHub propose deux branches (répertoires dépôts) : main ou master. Nos exemples sont construites sur la branche master.

La commande git branch vous permet de lister toutes les branches disponibles et met en évidence la branche active avec un astérisque (*) de votre compte GitHub.

1. Démarrer l'Invite de Commandes en Mode Administrateur

  1. Appuyez sur la touche Windows pour ouvrir le menu Démarrer.
  2. Tapez cmd dans la barre de recherche.
  3. Faites un clic droit sur Invite de commandes et sélectionnez Exécuter en tant qu'administrateur.

2. Vérifier la Version de Git

Assurez-vous que Git est installé et à jour en exécutant :

git --version

Pour mettre à jour Git :

git update-git-for-windows

3. Naviguer vers le Répertoire de Destination

Utilisez la commande cd pour vous déplacer vers le dossier où vous voulez cloner le dépôt :

cd C:\Utilisateurs\VotreNom\Documents\MesProjets

4. Cloner un Dépôt GitHub

Exécutez la commande suivante pour cloner un dépôt :

git clone https://github.com/Aranook/Apprendre-GitHub.git

Cela crée un dossier local contenant les fichiers du dépôt distant.

5. Naviguer dans le Dépôt Cloné

Accédez au dossier cloné avec :

cd Apprendre-GitHub

6. Ajouter et Valider des Modifications

  1. Ajoutez vos modifications au suivi avec :
    2. git add .
  2. Créez un commit :
    3. git commit -m "Description des modifications"

7. Pousser les Modifications vers GitHub

Envoyez vos modifications au dépôt distant :

git push origin main

Remplacez main par le nom de votre branche si nécessaire.

8. Mettre à Jour depuis le Dépôt Distant

Pour synchroniser votre dépôt local avec les dernières modifications du dépôt distant :

git pull origin main

9. Gérer les Branches

  1. Voir les branches disponibles :
    2. git branch
  2. Changer de branche :
    3. git checkout nom_de_branche

10. Utiliser une Adresse E-mail GitHub Privée

Configurez une adresse e-mail privée pour protéger votre e-mail personnel :

  1. Accédez à vos Paramètres GitHub.
  2. Cochez Keep my email addresses private.
  3. Utilisez l'adresse e-mail générée, par exemple : username@users.noreply.github.com.

Conseils Supplémentaires

  • Créez une nouvelle branche pour chaque fonctionnalité ou correctif majeur.
  • Utilisez des messages de commit clairs et descriptifs.
  • Synchronisez régulièrement votre dépôt local avec le dépôt distant.
