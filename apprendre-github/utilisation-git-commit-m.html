Introduction à git commit

La commande git commit est utilisée pour enregistrer les modifications dans l'historique du dépôt Git local. Chaque commit représente une version du projet à un instant donné, avec un message descriptif pour documenter les modifications.

Étape 1 : Ajouter des fichiers au suivi Git

Avant de créer un commit, vous devez ajouter les fichiers que vous souhaitez inclure dans ce commit. Utilisez la commande suivante pour ajouter tous les fichiers modifiés ou créés :

git add .

Pour ajouter un fichier spécifique, utilisez :

git add nom_du_fichier

Étape 2 : Créer un Commit

Une fois les fichiers ajoutés, vous pouvez créer un commit en utilisant la commande suivante :

git commit -m "premier commit"

Explication de la commande :

  • git commit : Indique que vous souhaitez enregistrer un ensemble de modifications.
  • -m : Spécifie que vous voulez inclure un message de commit directement dans la commande.
  • "premier commit" : Le message de commit. Il doit être descriptif et expliquer ce que vous avez fait dans ce commit.

Étape 3 : Vérifier les Commits

Pour vérifier que votre commit a bien été créé, utilisez la commande suivante :

git log

Cette commande affiche l'historique des commits. Vous devriez voir votre message de commit "premier commit" dans la liste.

Exemples de Messages de Commit

Voici quelques exemples de messages de commit clairs et concis :

  • git commit -m "Ajout du fichier README.md avec la description du projet"
  • git commit -m "Correction d'un bug dans la fonction calculTotal()"
  • git commit -m "Mise à jour du fichier .gitignore pour inclure les logs"

Bonnes Pratiques pour les Messages de Commit

  • Utilisez un message court mais descriptif.
  • Expliquez pourquoi une modification a été effectuée, pas seulement ce qui a été modifié.
  • Évitez les messages vagues comme "modifications" ou "update".

Conseils Supplémentaires

  • Pour modifier le dernier message de commit, utilisez : 
    git commit --amend -m "Nouveau message"
  • Si vous oubliez d'ajouter un fichier, vous pouvez l'ajouter et mettre à jour le commit avec --amend.
